❓ Frequently Asked Questions – Image to PDF

How does the auto-document detection work?
The camera uses the browser's video stream and analyzes each frame in real time on an HTML5 Canvas. It looks for rectangular regions with high contrast against the background — this is how it detects document edges. When a rectangle is found, a green bounding box appears and the camera auto-captures after 1.5 seconds. All processing happens entirely in your browser — no images are sent to any server.
What image formats are supported?
This tool supports JPG/JPEG, PNG, WebP and GIF images for upload. Images captured from the camera are processed as JPEG. Multiple images can be added to create a multi-page PDF — each image becomes one page in the PDF.
Can I reorder images before generating the PDF?
Yes. After adding images, you can drag and drop them in the image grid to reorder them. The order in the grid is the order they will appear in the final PDF. You can also remove individual images by clicking the red ✕ button on each image card.
What does the "B&W Doc" color preset do?
The "B&W Doc" preset applies a high-contrast black-and-white filter optimized for scanned documents and receipts. It increases contrast significantly, removes color saturation and enhances sharpness to make text crisp and readable. This is the best setting if you're scanning handwritten notes, printed documents or receipts that you want to archive as PDF.
Is my data private? Are images uploaded anywhere?
Yes, completely private. This tool works entirely in your browser using HTML5 Canvas API for image processing and a custom JavaScript PDF builder. Your images are never uploaded to any server, never stored and never shared. This makes it safe to use for confidential documents, ID cards, contracts and personal photos.
